addons: allow general parameters to be used when a format is not matched against
        formats are used when specific parameters should be in a specific order or
        when the input must meet a specified format/regex
  formats: user input matches specified regex string
           use the param names and their regex to allow matching input to be parsed into gui
    regex:  ^(?<paramName>\w)[ \.:](paramName2:[^.])$
            (?<paramName><type>]?)
    options: 
      admin: no|yes|defaultNo|defaultYes
      launch: web: $<url>?user=$<user>&pass=sha256()

  params
    name: name of parameter to show in gui
    type: str/int/float/date/time/datetime/regex/file/files/folder/fileList/folderList/fileFolderList
            fileList/folderList/fileFolderList allow user to add files/folder to the listview and perform an action on the files
    default value: 
      date/datetimes default to current day unless default is another value
